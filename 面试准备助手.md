# 面试准备助手 v2.0

## 概述

本 skill 帮助用户在面试前快速完成全面的准备工作。用户只需提供：
- **公司名称**（以及公司所在行业/产品方向，若已知）
- **岗位描述（JD）**
- **个人简历**（支持 .docx / .pdf / Markdown / 纯文本）

即可自动生成结构化的面试准备手册，**标准输出为 HTML 可视化网页（首选）+ MD 纯文本双版本**。

---

## 执行流程

### 第一步：收集输入

确认用户是否已提供以下三项内容：
1. 公司名称（+ 可选：主要产品/业务方向、所属事业群/部门）
2. 岗位描述（JD）全文
3. 个人简历（文件或文本均可）

若有缺失，逐项询问后再继续。

**简历解析规则：**
- `.docx` 文件：使用 python-docx 解析（需预装 `python-docx`），提取所有段落文本和表格内容
- `.pdf` 文件：使用 Read 工具直接读取
- Markdown / 纯文本：直接使用
- 若用户通过 BOSS直聘等平台分享了链接，**引导用户直接粘贴 JD 文本**（BOSS直聘有反爬机制，自动化提取不可靠）

---

### 第二步：公司层面调研

**目标：** 梳理公司业务全貌与真实口碑，帮助用户在面试中展现对公司的了解。

**执行步骤：**

1. 使用 `WebSearch` 工具，并行搜索以下维度的信息（每个维度至少一次搜索）：

   | 搜索维度 | 搜索关键词模板 |
   |----------|---------------|
   | 核心业务 | `{公司名} 主营业务 产品介绍` |
   | 官网 | `{公司名} 官网` → 用 `WebFetch` 抓取官网首页 |
   | 员工口碑 | `{公司名} 员工评价 面试体验` |
   | 产品口碑 | `{公司名} 产品评价 小红书 OR 知乎` |
   | 公司状况 | `{公司名} 公司怎么样 靠谱吗` |
   | 移动端产品 | `{公司名} App 评分 用户反馈`（若有） |
   | 近期动态 | `{公司名} 最新动态 融资 OR 新产品 2025 OR 2026` |
   | 创始人/团队 | `{公司名} 创始人 OR CEO OR 核心团队 背景` |
   | 竞品/行业 | `{公司名} 竞品 OR 行业分析` |

2. **深度调研技巧（基于实战经验积累）：**

   - **关联实体挖掘**：若发现公司有关联品牌、海外主体、子公司，务必进一步搜索。例如发现创始人另有海外品牌（如 Volvence），应搜索该品牌以理解公司真实定位。
   - **BOSS直聘/天眼查补充**：尝试用 WebFetch 访问 BOSS直聘公司页和天眼查/爱企查公司页，获取规模、融资、在招职位等结构化信息。
   - **产品体验**：若公司有移动端产品，搜索 `{产品名} App Store 评分` 或 `{产品名} 安卓 评价`，了解真实用户反馈。
   - **行业报告**：对于新兴赛道公司，搜索 `36kr {公司名} OR {赛道}` 或 `晚点LatePost {公司名}` 获取深度报道。

3. 汇总搜索结果，生成**公司调研报告**，结构如下：

```
## 公司调研报告

### 基本信息
- 公司全称 / 成立时间 / 规模（若可查）
- 融资情况 / 上市状态 / 所属集团（如腾讯SSV等）

### 核心业务与产品
- 主营方向（简短概括）
- 核心产品列表（含简介）
- 商业模式（若有线索）

### 团队背景
- 创始人/核心团队简介（若可查）

### 市场评价与口碑
- 正面评价（用户 / 员工视角）
- 负面评价 / 争议点
- 竞品对比（若搜到）

### 近期动态
- 重要新闻 / 战略变化 / 融资轮次等

### 面试前需了解的关键信息
- 3-5 条最重要的背景知识点，帮助候选人在面试中展示了解
```

---

### 第三步：JD 与简历匹配分析

**目标：** 找出用户简历与岗位要求的契合点和差距，输出量化匹配度评估。

**执行步骤：**

1. 仔细阅读 JD，提取：
   - **硬性要求**（学历、年限、特定技能、证书等，标注为"必须"）
   - **软性要求**（沟通能力、协作精神、数据思维等）
   - **加分项**（优先考虑但非必须的经验）

2. 对照用户简历，逐条判断匹配程度。使用以下量化标准：

   | 匹配等级 | 标签 | 判定标准 |
   |----------|------|----------|
   | 强匹配 | `极高` / `高` | 简历中有直接对应的项目/技能，数据可量化 |
   | 部分匹配 | `中` | 有相关经验但不完全符合，或需补充说明 |
   | 弱项/空白 | `中风险` / `高风险` | 无相关经验，需准备应对策略 |
   | 学历风险 | `中风险` | 学历不满足要求（但已进入面试说明对方放水） |

3. 生成**匹配分析报告**，结构如下：

```
## JD 与简历匹配分析

### 匹配度总览
- 整体匹配度：XX%（高 80%+ / 中高 60-80% / 中 45-60% / 低 <45%）
- 匹配度分布：强匹配 X% / 部分匹配 X% / 中风险 X% / 空白 X%

### 强匹配点（面试中重点展示）
| 等级 | 匹配维度 | 简历对应证据 |
|------|----------|-------------|
| 极高 | XX | XX |

### 部分匹配 / 需强化说明
| 等级 | 匹配维度 | 差距说明 | 补充建议 |
|------|----------|----------|----------|

### 风险项（短板）
| 等级 | 短板 | 应对策略 |
|------|------|----------|

### 总体评估
- 核心优势 1-2 句话总结
- 主要风险 1-2 句话说明
- 面试核心策略建议
```

---

### 第四步：面试题预测与回答建议

**目标：** 预测面试中可能出现的问题，并结合用户简历给出个性化的回答思路。

**执行步骤：**

1. 使用 `WebSearch` 工具，搜索该岗位的面试经验（并行搜索）：
   - `{岗位名称} 面试题 面经`
   - `{公司名} {岗位名称} 面试`
   - `{岗位名称} 常见面试问题 2025 OR 2026`
   - `{行业关键词} 面试 经验`

2. 整合搜索结果 + 第三步的匹配分析，生成**面试题预测报告**。题目分类及数量建议：

   | 分类 | 数量 | 说明 |
   |------|------|------|
   | 通用必考题 | 4-5 | 自我介绍、离职原因、职业规划、优缺点 |
   | 岗位专项题 | 6-8 | 与JD直接相关的专业问题 |
   | 行为面试题（STAR） | 3-4 | 团队协作、挑战经历、失败复盘 |
   | 公司/行业认知题 | 2-3 | 对公司业务的理解、行业趋势判断 |

3. 每道题的输出格式：

```
**Q：问题**
- **考察意图：** 面试官想了解什么
- **回答思路：** 基于用户简历的个性化建议（必须引用简历中的具体经历/数据）
- **参考要点：** 1-3 个关键要点
- **危险追问：** （若有）这道题可能引出的追问 + 应对策略
```

4. 额外生成**危险点汇总**（基于简历短板 + 行为面试可能暴露的弱点）：

```
### 可能被追问的"危险点"
| 风险点 | 可能追问 | 应对策略 |
|--------|----------|----------|
```

---

### 第五步：生成定制化自我介绍

**目标：** 生成一段适合该岗位、口语化、2分钟内能说完的自我介绍。

**执行要求：**
- 字数控制在 **250-300 字**（口语阅读约 1.5-2 分钟）
- 语气**自然口语化**，避免书面腔
- 结构：**个人背景 → 核心经验（匹配点） → 对该岗位的热情/理解 → 期待**
- 重点突出与 JD 最匹配的 2-3 段经历（含量化数据）
- 避免简历流水账，要有逻辑性和故事感
- **所有内容必须与用户简历保持一致，不得夸大或编造**

## 输出规范

### 标准交付物

每次执行必须输出**双版本**：

1. **HTML 可视化网页**（首选交付物）
2. **Markdown 纯文本**（备用/存档）

### 文件命名规则

- HTML：`面试准备_{公司名}_{岗位名}_{YYYYMMDD}.html`
- MD：`面试准备_{公司名}_{岗位名}_{YYYYMMDD}.md`

### HTML 可视化模板

必须遵循以下 HTML 结构和样式规范，确保：
- 移动端友好（响应式设计）
- 支持点击展开/折叠面试题回答
- 匹配度使用可视化图表（进度条、徽章等）
- 自我介绍区域醒目突出

**HTML 必须包含的模块（按顺序）：**

```
1. 顶部 Header（公司名 + 岗位名 + 日期）
2. 导航栏（锚点跳转：匹配分析 | 公司调研 | 面试题预测 | 危险点 | 自我介绍）
3. 匹配度总览卡片（数值 + 分布条 + 强匹配表 + 风险表）
4. 公司调研卡片（基本信息 + 核心业务 + 口碑 + 近期动态 + 面试须知）
5. 面试题预测卡片（分类折叠式：通用/专项/行为/行业）
6. 危险点汇总卡片
7. 自我介绍卡片（绿色高亮背景）
```

**CSS 设计规范：**
- 主题色：`#07c160`（绿色）作为品牌色
- 匹配徽章：绿色(强) / 橙色(中) / 红色(弱) / 蓝色(信息)
- 导航栏：`position: sticky` 固定在顶部
- 面试题：点击展开/折叠，使用 `max-height` 过渡动画
- 自我介绍：渐变绿色背景 `linear-gradient(135deg, #f0faf4, #e8f8ef)`
- 响应式断点：`@media (max-width: 600px)`

**参考模板文件：** 见 `references/html-template.html`

### MD 文本规范

Markdown 版本与 HTML 版本内容完全一致，使用标准 Markdown 语法：
- 一级标题：`# 面试准备手册`
- 二级标题：`## 公司调研` / `## 匹配分析` / `## 面试题预测` / `## 自我介绍`
- 表格用于匹配分析、风险项、面试题等结构化数据
- 加粗用于强调关键数据点和回答要点

## 注意事项

### 内容准确性
- 搜索过程中如遇敏感内容（如涉及政治），跳过该条目，不纳入报告
- 若某个搜索维度无有效结果，在报告对应章节注明"未找到相关公开信息"，不捏造内容
- 面试题回答建议**必须基于用户提供的简历内容**，切勿虚构经历
- 自我介绍中的所有内容必须与用户简历保持一致，不得夸大或编造

### 搜索策略经验（实战积累）
- BOSS直聘/招聘平台有反爬机制，**不要尝试自动化提取**，引导用户手动粘贴
- 对于创业公司/小型公司，务必搜索创始人背景和关联公司，往往能发现真实业务方向
- 若公司有海外品牌或子公司（如 .com 海外域名），搜索海外信息可能比国内信息更丰富
- 年份关键词使用当前年份及前一年（如 2025 OR 2026），覆盖最新动态

### 通用性
- 本 skill 不预设任何特定用户的简历信息，所有简历相关分析均基于用户当次提供的简历
- 不同用户使用时，简历解析和匹配分析结果自然不同
- 匹配度评估标准统一，但具体建议根据简历内容定制
